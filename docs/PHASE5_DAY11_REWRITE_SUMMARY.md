## 技術細節

### 使用的 Markdown 最佳實踐

1. **表格格式**:
   - 所有表格使用對齊（`|---------|`）
   - 欄位寬度一致
   - 表頭清晰

2. **程式碼區塊**:
   - 所有程式碼區塊標註語言（\`\`\`bash）
   - 註解使用繁體中文
   - 縮排一致（2 空格）

3. **連結格式**:
   - 使用相對路徑（`[文字](相對路徑.md)`）
   - 包含錨點（`[文字](檔案.md#章節)`）
   - 描述性連結文字（避免「點這裡」）

4. **列表格式**:
   - 有序列表使用數字
   - 無序列表使用 `-`
   - 巢狀列表縮排 2 空格

5. **強調格式**:
   - ✅ ❌ ⚠️ emoji 用於狀態
   - **粗體** 用於重要術語
   - `程式碼` 用於指令和配置
